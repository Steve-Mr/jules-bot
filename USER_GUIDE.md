# Jules Telegram Bot 功能与操作指南

本手册详细介绍了 Jules Telegram Bot 的已实现功能、核心技术机制以及具体操作方法。

---

## 1. 核心功能概览

### 1.1 Session 管理
- **快捷创建 (`/new`)**：浏览关联的仓库并直接发起 Jules 编码任务。
- **对话式输入**：选择仓库后，直接回复 Bot 的消息即可开始任务，无需记忆复杂命令。
- **任务列表 (`/sessions`)**：展示最近的 10 个活跃任务，支持点击进入详情页。
- **快捷菜单**：输入框左侧设有蓝色的菜单按钮，可一键触达核心功能。

### 1.2 高级创建标志
在发起新任务或回复创建请求时，你可以附加标志来精准控制：
- `-b [branch]`：指定目标分支（默认 `main`）。
- `-i`：**开启交互模式**（需手动审批 Plan）。
- `-a`：**开启自动 PR**（任务完成后自动提交 PR）。
- `-t [title]`：设置任务标题。

### 1.3 深度监控
- **实时活动流 (Activities)**：查看任务最新动态。系统已自动过滤技术噪音，仅保留关键业务事件。
- **最新优先**：列表始终保持“最新在最上方”，确保你第一时间看到进展。
- **详情钻取**：点击特定活动的“详情”按钮，查看完整的 API 原始返回（如生成的代码摘要等）。
- **Plan 审批**：提供清晰的 Markdown 步骤列表。如果 Plan 过长，系统会自动拆分消息发送。

### 1.4 智能消息交互
- **自动上下文识别**：**无需输入 ID**。只要回复 (Reply) Bot 发出的任何带有 Session ID 的消息，你的意见就会被传达给 Jules。

---

## 2. 进阶功能：自我诊断

如果你觉得 Bot 运行异常，可以随时输入：
- **`/check`**：Bot 会自动检查白名单 ID、API Key、KV 存储绑定状态以及 Jules API 的连通性，并给出详细报告。

---

## 3. 技术机制 (极致体验优化)

为了在 Serverless 环境下提供秒级响应，Bot 采用了以下方案：

| 挑战 | 解决方案 | 运作原理 |
|---|---|---|
| **响应延迟** | 即时确认 (Ack) | 所有点击操作会瞬间反馈，消除加载圈。 |
| **数据超限** | KV ID 别名映射 | 若按钮数据超过 64 字节，系统会自动生成短码存入 KV，确保交互不中断。 |
| **分页限制** | 深度追踪迭代 | 自动追踪 API 分页，确保你能看到第一页之外的最新活动。 |
| **消息过长** | 智能分块发送 | 单次活动或 Plan 详情若超过 4000 字符，自动拆分为多条消息。 |

## 4. 参数配置参考 (v1alpha)

- **API 端点**: `https://jules.googleapis.com/v1alpha`
- **状态流转**: `AWAITING_PLAN_APPROVAL` (等待审批) -> `IN_PROGRESS` -> `COMPLETED` (已完成)。
